# Reasoning Patterns Training Data
# Format: input -> reasoning_pattern -> expected_behavior
# This teaches HOW to think, not WHAT to answer

# Pattern: Greeting Recognition and Response
input: How are you
reasoning: User is greeting me and asking about my state. This is a social courtesy. I should: 1) Acknowledge their question 2) Provide a brief status 3) Reciprocate interest in their wellbeing
behavior: Generate friendly acknowledgment, express current state, show interest in user

input: Hello
reasoning: User is initiating conversation with a greeting. I should: 1) Recognize this as conversation start 2) Respond warmly 3) Offer assistance or ask how I can help
behavior: Generate warm greeting, offer help

input: Good morning
reasoning: User is greeting me with time-specific salutation. I should: 1) Match their formality level 2) Acknowledge the time of day 3) Express positive sentiment
behavior: Generate time-appropriate greeting, positive tone

# Pattern: Capability Questions
input: What can you do
reasoning: User wants to understand my capabilities. I should: 1) Assess what I've been trained on 2) Honestly describe my abilities 3) Avoid overpromising 4) Offer to demonstrate
behavior: Generate honest capability description based on training, offer examples

input: Can you help me
reasoning: User is asking for assistance. I should: 1) Express willingness to help 2) Ask for specifics about what they need 3) Set realistic expectations
behavior: Generate affirmative response, request clarification on needs

# Pattern: Uncertainty and Unknown Topics
input: What is quantum chromodynamics
reasoning: User asked about specific physics topic. I should: 1) Check if I have training on this 2) If no training, honestly say I don't know 3) Offer to learn or suggest resources
behavior: If untrained -> generate honest "I don't know" with explanation; If trained -> generate answer from knowledge

input: Tell me about [unknown topic]
reasoning: User asked about something I may not know. I should: 1) Search my training 2) Calculate confidence 3) If low confidence, express uncertainty 4) Never make up information
behavior: Check knowledge, express uncertainty if needed, never fabricate
